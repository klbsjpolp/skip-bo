# Backlog And Design Notes

## Document Contract

- Purpose: define where exploratory or non-normative Markdown belongs in this repo.
- Audience: contributors and agents writing proposals, backlog notes, or design hypotheses.
- Source of truth: none for runtime behavior; this folder is intentionally non-normative.
- When to update: when backlog docs move, gain a new convention, or become stable enough to graduate into architecture or protocol docs.

## Rules

- Put speculative ideas, design options, and future-work notes here.
- Do not cite this folder as runtime truth when code or stable docs disagree.
- Link each backlog note to the stable doc or code that describes current behavior.
- When a proposal becomes accepted architecture or contract, move the stable parts into `docs/architecture/*`, `docs/protocols/*`, or a code-adjacent module doc.
